# 分け目ピン機能 現状レポート（Codexレビュー依頼用）

作成: 2026-07-11 / 対象: `index.html`（branch `shell-blend`、最終コミット 25389d7 の上の未コミット変更）

## 背景

このフォークは 852wa/Anime2.5DRig の首振りを3D楕円体シェル投影に拡張したもの。前髪は専用の HAIR シェル（HEADの1.10倍）に乗り、`前髪の浮き`(hairGap) 分だけ額から浮いて回転する設計。前髪を下ろしたキャラでは問題ないが、新サンプル（デコ見せ・前髪分け目あり）では回転時に前髪の縁が額の地肌からズレて浮いて見える。

これを直すため「分け目ピン」を実装した: 前髪メッシュの頂点ごとに分け目基準点からの距離で重み wR を計算し、根元（wR≈1）は顔と同じ HEAD シェル投影に密着、毛先（wR≈0）は従来の浮いた HAIR シェルのままブレンドする。

## 実装箇所（index.html）

- UI: 前髪セクションにトグル `分け目あり`(tgPart→auto.part, localStorage `a25d_part` に永続化)・`分け目を表示`(tgPartViz)、スライダー `分け目の位置`(partX, -1..1, 0)・`根元の密着`(rootPin, 0..1, 初期1)・`密着範囲`(rootR, 0.1..1.5, 初期0.9)
- L1070前後: deform() のホイスト。分け目基準点 = `(HEAD.cx + partX*HEAD.rx*0.9, A.face.y0 + 顔高さ*0.05)`（生え際中央）。`doPin = isFH && auto.part && rootPin>0.001`
- L1135前後: 頂点ループ内の重み計算:
  `pdd = hypot((x-partPx)/HEAD.rx, max(0, y-partPy)/HEAD.ry)`（生え際より上=頭皮は全ピン、下だけ減衰）
  `wR = rootPin * smoothstep((rootR-pdd)/rootR)`
- A系統（レガシー視差）: 実効深度 `dd = L.depth + (1-L.depth)*wR` で前髪深度1.28→顔1.0へ寄せる
- B系統（シェル投影）: 頂点を顔と同じHEADシェル（gap/bulgeなし、横顔カーブ追従）で再投影し、**差分方式**（頭皮投影の回転デルタを前髪の静止投影に加算）で正面時の恒等性を厳密に保証。`sx += (tx-sx)*wR`
- L1461前後: デバッグ表示（マゼンタの基準点ドット＋影響範囲境界線、前髪ワイヤーフレーム橙）
- L779: テストフック `window.__pinProbe(angleX, partOn, tx, ty, overrides)` — rAF不要で deform() を直接実行し、指定座標に最も近い前髪頂点の変形後座標を返す

## ユーザー報告の症状

「密着範囲(rootR)スライダーを弄っても何も変わらない感じがする」（分け目ありON・角度Xを振った状態で目視）。

## こちらの見立て（要検証）

数値上スライダーは効いているが、**知覚的に見えない**のが実態と思われる:

1. 頭部全体の移動量（50〜100px規模）に対し、ピンの寄与が最大でも15px程度。特にrootRを全域動かしても生え際中央は11.7→14.9pxしか変わらず、変化率が小さすぎる
2. ユーザーが注視しているのはおそらく「額の左右を縁取る前髪エッジ（こめかみ付近）」で、そこはrootR=1.5まで上げても7.8px。初期値0.9では1px
3. smoothstep減衰が急で、境界付近の細い帯でしか差が出ない

## Codexに確認・提案してほしいこと

1. **知覚可能性の検証**: 上記実装で「デコ見せの分け目が破綻しない」という目的は達成できているか。生え際15px/こめかみ8pxの引き戻しは十分か、それとも設計自体（点基準の距離減衰）を変えるべきか
2. **減衰モデルの代替案**: こめかみの縁まで自然に密着させるには、(a) 分け目点からの距離ではなく「生え際アーチ（HEAD楕円上縁）からの距離」で減衰させる、(b) ストランド位置 L.su（毛の根元0→毛先1）を重みに使う、(c) 前髪レイヤーのアルファから根元マップを自動生成する、などのどれが筋がよいか
3. **バグの可能性**: rootRの効きが頂点メッシュ解像度（セル約42px）で量子化されて見えにくい、などの実装起因の問題はないか。`__pinProbe` で追試可能
4. **UXの改善**: 「スライダーが効いてない感」を消すには、範囲変更時に影響領域を一瞬ハイライトする、rootRの意味を変える（例: こめかみ基準）などの案はあるか

## 再現手順

1. `python -m http.server 8931` でリポジトリ直下を配信、`http://localhost:8931/` を開く
2. 「sample.psd を読み込む」→ 前髪セクションで「分け目あり」「分け目を表示」をON
3. 角度Xを±1に振りながら 密着範囲 0.1↔1.5 を往復して目視
4. 数値検証はDevToolsで `__pinProbe(1,false,500,270)` と `__pinProbe(1,true,500,270)` の出力座標を比較（第3・4引数=注目点、第5引数で `{rootR:1.5}` 等の上書き可）
